02

Systems Design

Async vs Synchronous Execution

Move work off the user's critical path. The fraud service doesn't wait to be queried at transaction time — it runs in the background and caches results downstream as signals arrive. By execution time, the answer is already there. The user never waits for work the system could have done earlier.

Performance Architecture

Thread Utilisation & Server Sizing

A process with modest throughput requirements gets a single-core instance. A service handling high call volume with latency sensitivity gets parallelised, and the server gets sized to match. The software and the hardware were designed together, not independently.
